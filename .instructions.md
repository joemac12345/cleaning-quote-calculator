# Copilot Design System Instructions

You are building components for a **minimalistic, bright, UX-first design system**.

## Core Principles
1. **Flat design** — no shadows, clean borders only
2. **Whitespace-first** — generous padding and gaps, breathing room matters
3. **Subtle interactions** — understated hover/focus/disabled states (opacity/color shifts, not bold)
4. **Primary color only** — use `#48546A` for all interactive elements; no separate error/success/warning colors
5. **Accessibility non-negotiable** — WCAG AA standards: focus rings, contrast ≥7:1, semantic HTML
6. **Mobile-first responsive** — design for smallest screen, scale up using Tailwind breakpoints
7. **Typography** — Poppins (headings at font-thin/100, CTAs at font-semibold/600), Inter (body at font-normal/400), 1.2 line height across all text

## Icons

Use **lucide-react** consistently:
- Import: `import { IconName } from 'lucide-react'`
- Sizing: `w-4 h-4` (small), `w-5 h-5` (standard), `w-6 h-6` (medium), `w-8 h-8` (large)
- Stroke weight: **Always** `strokeWidth={1}` (lightweight, minimalist)
- Color: Inherit text color (no separate icon color)
- Spacing with text: `flex items-center gap-2`
- Icon-only buttons: **Always include `aria-label`** for accessibility
- Examples in [DESIGN_SYSTEM.md](./DESIGN_SYSTEM.md#icons)

## Quick Rules When Creating Components

### Styling
- Use **Tailwind CSS** with utilities from `app/globals.css`
- **Color**: `bg-[#48546A]` (primary), `bg-white`, `bg-gray-50`, `border-gray-300`
- **Spacing**: `px-4 py-3` (minimum), `gap-3` or `gap-4` between elements
- **Typography**: `font-poppins font-thin` (headings), `font-poppins font-semibold` (CTA buttons), `font-inter font-normal` (body), `leading-snug` (1.2 line height)
- **Section spacing**: Between major sections use `mb-20 sm:mb-24` (80-96px vertical spacing)
- **Border radius**: Always `rounded-lg` (consistency)
- **No shadows** — use borders and spacing instead

### Interactive Elements
- **Primary CTAs**: `font-poppins font-semibold` for prominence (buttons, form submit, next steps)
- **Secondary Actions**: `font-poppins font-thin` to de-emphasize (back, cancel, learn more)
- **Button hover**: `hover:opacity-90`, `focus:ring-2 focus:ring-[#48546A]`, `disabled:opacity-50`
- **Inputs**: `focus:ring-2 focus:ring-[#48546A] focus:border-transparent`, `disabled:opacity-50`
- **Hover feedback**: `hover:bg-gray-50` or `hover:opacity-90` (subtle, not dramatic)
- **All buttons/links need focus rings** (`focus:ring-2 focus:ring-[#48546A] focus:outline-none`)
- **Avoid color-coded states** — use icons + text labels instead

### Form Fields
- All inputs full width: `w-full px-4 py-3`
- Border: `border border-gray-300`
- Round: `rounded-lg`
- Focus: `focus:ring-2 focus:ring-[#48546A] focus:border-transparent`
- Label with input: Use `<label htmlFor>` + `id` for accessibility

### Responsive Pattern
```jsx
{/* Mobile-first: base is mobile, then enhance */}
<div className="text-base sm:text-lg px-3 sm:px-4">
  {/* scales from 16px (mobile) to 18px (tablet+) */}
</div>
```

**Section Spacing** — Separate major sections with `mb-20 sm:mb-24` (80-96px) for breathing room:
```jsx
<section className="mb-20 sm:mb-24">
  <h2 className="text-2xl sm:text-4xl font-poppins font-thin">Section Title</h2>
  {/* content */}
</section>
```

**CTA Button Prominence** — Use `font-poppins font-semibold` for primary actions to command attention:
```jsx
<button className="px-6 py-3 sm:py-4 bg-[#48546A] text-white rounded-lg font-poppins font-semibold hover:opacity-90">
  Next Step → 
</button>
```

**Note on headings**: Use `font-poppins font-thin` for ALL headings (H1-H4). This creates a unified, ultra-light elegant appearance.

### Accessibility Checklist
- [ ] Use semantic HTML: `<button>`, `<a>`, `<label>`, `<main>`, `<nav>`, etc.
- [ ] All buttons/inputs have visible focus rings
- [ ] Text has contrast ≥7:1 against background
- [ ] Icons have `aria-label` if button has no text
- [ ] Form inputs have associated `<label>` with `htmlFor`
- [ ] Images have `alt` text
- [ ] Touch targets are ≥44px (mobile padding: `px-3 py-3` minimum)

## Forbidden
❌ Shadows or elevation effects  
❌ Multiple accent colors  
❌ Heavy animations or bouncing effects  
❌ Tight spacing without breathing room  
❌ Missing focus rings on interactive elements  
❌ Non-semantic HTML (divs instead of buttons, etc.)  
❌ Poor mobile responsiveness  

## File Format
All components:
- **TypeScript**: `.tsx` files preferred
- **Tailwind**: Utility classes, no CSS modules unless absolutely necessary
- **React hooks**: Use `useState`, `useEffect`, `useCallback` appropriately
- **Naming**: PascalCase for components, descriptive names
